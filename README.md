# Factory Distance Tool

一个本地运行的厂区设备与厂界距离测算工具。

- 使用 `Python` 启动本地 HTTP 服务
- 使用浏览器页面完成图片标注、距离计算、工程保存与导出
- 不依赖后端 API，也不需要独立数据库服务
- 适合在厂区截图、总平面图上标注设备点并测算到厂界的距离

## 功能特性

- 上传 `PNG/JPG` 厂区截图作为底图
- 通过两点定义比例尺，建立像素与真实距离换算关系
- 设置坐标原点，输出设备相对坐标
- 绘制东、南、西、北四个方向厂界，且每个方向支持多段线
- 添加设备点并查看到四向厂界的最短距离
- 支持录入建筑物名称、声源名称、型号、声功率级、控制措施、运行时段等噪声参数
- 根据距室内边界距离自动计算室内边界声级 dB(A)
- 支持建筑物插入损失和建筑物外距离，并自动计算建筑物外噪声声压级 dB(A)
- 支持使用多点描绘建筑物，并在左侧列表中编辑建筑物名称与插入损失
- 点击设备点可弹出编辑信息，更新型号、声功率级、运行时段和所属建筑物
- 比例尺改为隐藏式设置面板，并支持多主题颜色切换
- 设备和建筑物参数改为网页中部弹出式表单，不再使用浏览器默认弹窗
- 清空类操作改为网页中部二次确认对话框
- 新增撤销后的“恢复撤销”能力，避免误撤销
- 建筑物支持顶点编辑模式，可拖拽和删除单个顶点
- 设备支持单击选中、双击编辑、复制、删除和拖拽移动
- 自动保存到浏览器 `IndexedDB`
- 导入/导出工程 `JSON`
- 导出设备数据 `CSV`

## 工作原理

应用整体是一个“单文件本地工具”：

1. 运行 `app.py`
2. Python 启动本地 HTTP 服务
3. 浏览器加载内嵌的单页应用
4. 用户在 Canvas 上完成比例尺、原点、厂界、设备点标注
5. 前端直接完成几何计算、自动保存、工程导入导出

距离计算基于“点到线段最短距离”算法。

## `app.py` 模块说明

### 前端页面模板

`app.py` 中的 `HTML` 常量内嵌了完整单页应用，包含：

- 页面布局与控件
- 视觉样式
- 前端状态管理
- Canvas 绘制逻辑
- 工程导入导出与自动保存

### 前端状态与数据规范化

前端维护一个 `state` 对象，负责保存：

- 当前图片与工程名称
- 当前操作模式
- 比例尺、坐标原点
- 四个方向的厂界线段
- 设备点列表
- 撤销历史与自动保存时间

同时提供一组 `normalize*` 方法，确保导入工程、恢复自动保存时的数据兼容与安全。

### 自动保存与工程序列化

核心逻辑包括：

- `buildProjectPayload()`：把当前工程组织成可保存数据
- `saveAutosaveNow()`：写入 `IndexedDB`
- `loadAutosaveProject()`：恢复自动保存工程
- `exportProjectFile()` / `importProjectFile()`：导出和导入工程文件

工程文件内会保存图片的 `data URL`，因此是自包含的。

### 绘图与几何计算

Canvas 负责：

- 绘制底图、比例尺、原点、厂界、设备点
- 根据鼠标状态显示临时辅助线
- 按“点到线段最短距离”算法计算设备到厂界距离

关键函数包括：

- `pointToSegmentDistance()`
- `pixelToActual()`
- `getBoundaryDistanceValue()`

### 本地服务层

Python 端非常轻量：

- `SinglePageHandler`：仅返回首页 HTML
- `main()`：解析命令行参数、启动本地服务、自动打开浏览器

## 运行方式

### 直接运行 Python

```bash
python app.py
```

默认启动本地服务后自动打开浏览器。

可选参数示例：

```bash
python app.py --host 127.0.0.1 --port 8765 --no-browser
```

## Windows 打包

双击运行：

- `build_windows.bat`

脚本会自动：

- 创建 `.venv-build`
- 安装 `PyInstaller`
- 生成 `dist\FactoryDistanceTool.exe`

## 已知注意点

- 当前没有 `requirements.txt` 或 `pyproject.toml`
- `WINDOWS_EXE_README.md` 中的默认监听地址与源码默认值有轻微不一致，源码当前默认 `--host 0.0.0.0`
- 前端页面、业务逻辑、服务端代码目前都集中在 `app.py` 中，后续功能增多后维护成本会上升
